# Diário de Atividades - Rafael Del Antonio (PO & Dev)

Este documento descreve as atividades realizadas durante o planejamento do sistema SafeCity Map, atuando como Product Owner e Desenvolvedor.

---

## Atividades como Product Owner

### 01. Definir funcionalidades principais
* **Descrição:** Organizei as principais funcionalidades do sistema como mapa interativo, cadastro de riscos e sistema de denúncias.
* **Objetivo:** Definir o escopo inicial do projeto.

### 02. Criar quadro Kanban no Trello
* **Descrição:** Criei o quadro Kanban com colunas Backlog, A Fazer, Fazendo e Feito.
* **Objetivo:** Organizar o fluxo de trabalho da equipe.

### 03. Definir categorias de risco
* **Descrição:** Classifiquei os riscos em categorias como criminalidade, acidentes e desastres naturais.
* **Objetivo:** Facilitar a organização das informações.

### 04. Definir padrão de cores para riscos
* **Descrição:** Defini cores para cada nível de risco (vermelho, laranja, amarelo e verde).
* **Objetivo:** Melhorar a visualização no mapa.

### 05. Definir modelo de cadastro de cidades
* **Descrição:** Estruturei os dados como nome, estado, país e localização.
* **Objetivo:** Padronizar os dados do sistema.

### 06. Planejar mapa interativo
* **Descrição:** Defini como os riscos serão exibidos no mapa.
* **Objetivo:** Garantir boa experiência do usuário.

### 07. Planejar integração com APIs de mapas
* **Descrição:** Analisei o uso de APIs para exibir mapas e localização.
* **Objetivo:** Permitir geolocalização no sistema.

### 08. Definir permissões de usuário
* **Descrição:** Separei usuários em comum e administrador.
* **Objetivo:** Garantir controle de acesso.

### 09. Planejar cadastro de novos riscos
* **Descrição:** Estruturei como usuários poderão registrar riscos.
* **Objetivo:** Permitir participação da comunidade.

### 10. Planejar sistema de busca
* **Descrição:** Defini a busca por cidade no sistema.
* **Objetivo:** Facilitar navegação.

### 11. Definir histórico de ocorrências
* **Descrição:** Planejei armazenamento de registros antigos.
* **Objetivo:** Permitir análise futura.

### 12. Planejar dashboard
* **Descrição:** Defini exibição de estatísticas e gráficos.
* **Objetivo:** Melhorar análise de dados.

### 13. Planejar atualização de dados
* **Descrição:** Estruturei como os dados serão atualizados.
* **Objetivo:** Manter sistema atualizado.

### 14. Planejar listagem de riscos
* **Descrição:** Defini tela com todos os riscos cadastrados.
* **Objetivo:** Melhor visualização.
